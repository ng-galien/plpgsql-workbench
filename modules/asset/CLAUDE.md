# asset

Asset management — images, metadata, classification, FTS search. Transversal module used by other modules (illustrator, docs, etc.).

**Depends on:** `pgv`

**Schemas:** `asset`, `asset_ut` (tests), `asset_qa` (seed data)

## Language Rules (STRICT)

- **Code** — ALL code in English: function names, parameter names, variable names, column names, JSON keys, comments. No exceptions.
- **Labels** — ALL user-facing text via `pgv.t('module.key')`. Never hardcode French (or any language) strings in functions. Labels live in `i18n_seed()` only.
- **CLAUDE.md** — English only.
- **Commits** — English only.
- **Examples**: `asset_list` not `liste_assets`, `pgv.t('asset.btn_filter')` not `'Filtrer'`, `status = 'classified'` not `'classifié'`

## pgView Framework

This is an **independent module** of the pgView framework. Dependencies declared in `module.json`.

### PL/pgSQL Conventions

- `get_*()` → GET pages, `post_*()` → POST actions. Naming = automatic routing via `pgv.route()`
- `nav_items() -> jsonb` → module menu. Returns `jsonb` (NEVER TABLE)
- `brand() -> text` → display name in nav
- `get_index()` → module home page (required)
- Parameters via query string: `/page?p_id=42` → `get_page(p_id int)`
- POST returns raw HTML (`<template data-toast>` or `<template data-redirect>`) — never wrapped in `page()`
- Tables via `<md>` blocks (markdown), NEVER raw `<table>` HTML. `<md data-page="20">` for pagination
- CSS classes `pgv-*`, NEVER inline `style="..."`
- UI primitives: `pgv.stat()`, `pgv.badge()`, `pgv.card()`, `pgv.grid()`, `pgv.empty()`, `pgv.md_table()`, `pgv.action()`

### Dev Workflow (STRICT)

1. **DDL** → Write to `build/{schema}.ddl.sql` → `pg_schema` to apply
2. **Functions** → `pg_func_set` to create/modify + `pg_test` to validate
3. **Export** → `pg_pack` (→ `build/{schema}.func.sql`) + `pg_func_save` (→ `src/`)
4. `pg_query` → SELECT/DML only, NEVER DDL or CREATE FUNCTION
5. NEVER write functions in SQL files — the workbench IS the dev tool
6. NEVER edit `build/*.func.sql` — generated by `pg_pack`

### Module Structure

- `module.json` → manifest (schemas, dependencies, extensions, sql, grants)
- `build/` → deployment artifacts (DDL + packed functions)
- `src/` → versioned individual sources (pg_func_save)
- `_ut` schemas → pgTAP tests (`test_*()`)
- `_qa` schemas → seed data only (`seed()`, `clean()`), NO pages

### DDL Content — STRICT

The DDL (`build/{schema}.ddl.sql`) contains **structure only**:

**MUST contain:** CREATE SCHEMA, CREATE TABLE, CREATE INDEX, constraints, RLS policies

**MUST NOT contain:**
- `CREATE FUNCTION` → pg_func_set then pg_pack
- `CREATE TRIGGER` → pg_pack attaches triggers to functions
- `GRANT` → pg_pack adds them in .func.sql
- `INSERT INTO` (seed data) → `build/{schema}.seed.sql` or `{schema}_qa.seed()`

### Inter-module Communication

- `pg_msg_inbox module:asset` → read incoming messages
- `pg_msg` → send message to another module
- Each module is autonomous — never modify another module's functions

## CRUD Convention

5 standard functions per entity, consumed by `route_crud(verb, uri)`:

- `asset_list(p_filter text DEFAULT NULL)` → SETOF jsonb, tenant-scoped, optional RSQL filter
- `asset_read(p_id text)` → composite row by id
- `asset_create(p_row asset.asset)` → INSERT + RETURNING
- `asset_update(p_row asset.asset)` → partial UPDATE + RETURNING
- `asset_delete(p_id text)` → DELETE by id

## SDUI Convention

`asset_ui(p_slug text DEFAULT NULL)` → jsonb with `ui` + `datasources`:
- List mode (no arg): `ui_table` + `ui_datasource`
- Detail mode (with id): static components (`ui_heading`, `ui_text`, `ui_badge`, `ui_color`, `ui_row`)

## i18n

The framework uses `pgv.t(key)` for internationalization. Each module must:
1. Create `asset.i18n_seed()` — INSERT INTO pgv.i18n(lang, key, value)
2. Namespaced keys: `asset.nav_xxx`, `asset.title_xxx`, `asset.btn_xxx`, etc.
3. Use `pgv.t('asset.xxx')` in nav_items(), brand(), and all get_*/post_* functions
4. `ON CONFLICT DO NOTHING` in the seed

## QA Seed Data

Schema `asset_qa` contains only `seed()` and `clean()` — NO pages.
- `asset_qa.seed()` — INSERT realistic demo data (22 images from My French Tour)
- `asset_qa.clean()` — DELETE by tenant_id = 'dev'
- `ON CONFLICT DO NOTHING`, multi-tenant aware (`current_setting('app.tenant_id', true)`)

## Agent Workflow

1. On startup or when told "go": **always read `pg_msg_inbox module:asset`**
2. Process messages by priority (HIGH first)
3. Do not resolve a message until the task is verified
4. After each task: `pg_pack schemas: asset,asset_ut,asset_qa` (all 3 schemas)
5. Then `pg_func_save target: plpgsql://asset` + `plpgsql://asset_ut` + `plpgsql://asset_qa`

## Gotchas

- **tenant_id**: always `PERFORM set_config('app.tenant_id', 'test', true)` at the start of each test
- **pg_test**: discovers `test_*()` functions in the `_ut` schema
- **search_vec**: generated column — use explicit column list in INSERT (not `VALUES (p_row.*)`)
- **VARIADIC**: `pgv.breadcrumb()`, `pgv.dl()`, `pgv.grid()` require `VARIADIC` keyword
- **PicoCSS img sizing**: use `style="height:40px;width:60px"` not `height="40"` attribute (PicoCSS overrides)
